# 福祉通知ウィキ 運用引き継ぎ

## 1. 概要

- アプリ名: 福祉通知ウィキ
- ローカル作業フォルダ: `C:\welfare-notice-wiki`
- GitHub: `https://github.com/AUL-DoX/welfare-notice-wiki`
- Vercel標準URL: `https://welfare-notice-wiki.vercel.app`
- 本番URL: `https://wn-wiki.aul-dox.jp`

このアプリは、障害福祉・介護関係の通知文書を一覧化し、検索・詳細表示できる Next.js アプリです。

運用方針は以下です。

- 一般利用者は閲覧専用
- 管理者のみ資料追加、カテゴリ設定、タグ設定を行う
- 本番公開は Vercel
- データ反映は GitHub に push して行う

## 2. 現在の構成

### 2-1. 文書データ

- 元資料置き場: `C:\welfare-notice-wiki\source-docs`
- 対応拡張子: PDF / PPTX / TXT / MD / CSV

ここに置いたファイルが、アプリの文書一覧・検索対象になります。

### 2-2. 設定データ

- カテゴリ設定: `C:\welfare-notice-wiki\data\document-categories.json`
- 手動キーワード設定: `C:\welfare-notice-wiki\data\document-keywords.json`
- アップロード日メタデータ: `C:\welfare-notice-wiki\data\document-metadata.json`

### 2-3. コード

- 画面本体: `C:\welfare-notice-wiki\src`
- 文書読込ロジック: `C:\welfare-notice-wiki\src\lib\documents.ts`

## 3. 日常運用

### 3-1. 資料を追加する

1. `source-docs` に資料を追加する
2. ローカルで確認する
3. 問題なければ GitHub に push する
4. Vercel の自動再デプロイを待つ
5. 本番URLで確認する

### 3-2. ローカル確認

PowerShell で以下を実行します。

```powershell
cd C:\welfare-notice-wiki
npm.cmd run dev
```

ブラウザで以下を開きます。

- `http://localhost:3000`

### 3-3. 本番反映

```powershell
cd C:\welfare-notice-wiki
git add .
git commit -m "更新内容"
git push
```

push 後、Vercel が自動再デプロイします。

## 4. カテゴリ設定

カテゴリは以下のJSONで管理します。

- `C:\welfare-notice-wiki\data\document-categories.json`

形式:

```json
{
  "資料のslug": "disability"
}
```

使用カテゴリ:

- `unclassified`
- `care`
- `disability`
- `common`

### 4-1. 注意

本番の Vercel は実行中ファイル書き込みができません。

そのため、カテゴリ変更は以下のどちらかで行います。

- ローカルJSONを編集して `git push`
- 管理者モードをローカルで使い、保存後に `git push`

## 5. タグ / 関連キーワード設定

手動キーワードは以下のJSONで管理します。

- `C:\welfare-notice-wiki\data\document-keywords.json`

形式:

```json
{
  "資料のslug": {
    "manualKeywords": [
      "障害福祉",
      "処遇改善",
      "賃上げ支援"
    ]
  }
}
```

### 5-1. 登録ルール

- `#` は保存時には不要
- 同じタグが別資料で重複しても問題なし
- 1資料内の重複は避ける
- 基本は ChatGPT などで候補を作ってから貼り付ける運用でよい

### 5-2. 反映方法

JSON編集後に以下を実行します。

```powershell
cd C:\welfare-notice-wiki
git add data/document-keywords.json
git commit -m "Add manual keywords"
git push
```

## 6. アップロード日

画面上の日付は、資料本文の日付やファイル更新日ではなく、アップロード日として扱っています。

保存先:

- `C:\welfare-notice-wiki\data\document-metadata.json`

新しいファイルを追加したときは、このメタデータが更新対象になります。

## 7. 管理者モード

### 7-1. 目的

管理者だけが、カテゴリ変更やタグ保存UIを使えるようにするための簡易モードです。

### 7-2. 仕組み

- 環境変数 `ADMIN_MODE_TOKEN` を設定する
- URLに `?admin=トークン` を付けて開く

例:

```text
https://wn-wiki.aul-dox.jp/docs/資料slug?admin=秘密の文字列
```

### 7-3. 設定場所

ローカル:

- `C:\welfare-notice-wiki\.env.local`

```env
ADMIN_MODE_TOKEN=秘密の文字列
```

Vercel:

- Project Settings
- Environment Variables
- `ADMIN_MODE_TOKEN` を追加

### 7-4. 注意

これは簡易的な管理者モードです。

- URLにトークンが見える
- 本格的なログイン認証ではない

そのため、厳密な認証が必要な用途には向きません。

## 8. ドメイン / DNS

現在の本番URL:

- `https://wn-wiki.aul-dox.jp`

DNSの実管理は **お名前.com 側** です。

重要事項:

- さくら側の DNS を触っても本番へ反映されない場合がある
- 実際の権威ネームサーバーは `01.dnsv.jp` 系
- ドメイン変更時はまずお名前.com側を確認する

## 9. Vercel運用

### 9-1. デプロイ確認

Vercel の `Deployments` で確認します。

状態:

- `Building`
- `Ready`
- `Error`

`Ready` になれば本番反映完了です。

### 9-2. よくある注意

Vercel本番では、実行中にローカルファイルへ書き込みできません。

そのため以下は本番では直接保存できません。

- JSONファイルへの恒久保存
- ローカルファイル更新を伴う管理操作

本番で編集したい場合は、将来的には DB や外部保存先が必要です。

## 10. 今後の推奨運用

現時点では以下の運用が最も安定です。

1. 資料を `source-docs` に追加
2. ChatGPT でタグ候補を作る
3. `document-keywords.json` に追加
4. 必要なら `document-categories.json` を修正
5. ローカルで確認
6. `git push`
7. 本番確認

## 11. トラブル時の切り分け

### 本番だけおかしい

- Vercel の最新デプロイが `Ready` か確認
- `git push` が完了しているか確認

### 独自ドメインだけおかしい

- お名前.com 側の DNS を確認
- `wn-wiki` の CNAME 設定を確認

### ローカルでは見えるが本番に出ない

- GitHub に push しているか確認
- Vercel が最新コミットを取り込んでいるか確認

### タグやカテゴリが保存されない

- Vercel本番では保存できない場合がある
- ローカルJSONを編集して `git push` する

## 12. 現在の認識

- アプリは公開閲覧用として運用
- 管理作業は基本的に管理者のみ
- 閲覧者にカテゴリやタグを触らせる想定はない
- そのため、今後は「管理者専用モード」または「ローカルJSON編集」を中心に使う
